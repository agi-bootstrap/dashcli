## Vision

### 10x Check
The current plan creates a CLI that generates dashboards. The 10x version creates an **autonomous data analyst** that:
- Connects to your data warehouse and proactively discovers interesting patterns
- Creates, updates, and retires dashboards as underlying data changes
- Answers follow-up questions about dashboards by querying data and generating explanatory charts
- Learns your team's dashboard style over time
- Acts as a dashboard API — other tools and agents can request dashboards via MCP

The current plan is a tool. The 10x version is a teammate.

### Platonic Ideal
"I told my terminal I needed to understand revenue by region. Thirty seconds later, a link opened in my browser with a beautiful, interactive dashboard that looked like our design system. I shared the link with my VP. She clicked a chart and asked 'why did APAC spike?' — and the dashboard answered her question by generating a drill-down on the fly. My VP has never opened Metabase since."

The emotional experience: magic that respects your intelligence. Not flashy demo-ware — something that a senior data scientist trusts enough to share with executives.

## Accepted Scope (added to this plan)
- Smart Schema Summary: auto-introspect tables, row counts, column types, sample values, relationships. Runs implicitly during `dashcli create --source ...` (no separate connect command needed)
- `dashcli theme`: 4 built-in themes (corporate, dark, minimal, colorful), reads DESIGN.md if present
- `dashcli watch` / `--watch` flag: file watcher + WebSocket auto-refresh on spec changes

## Architecture Decisions (from Sections 1-11)

| # | Decision | Rationale |
|---|----------|-----------|
| A1 | TypeScript + Bun runtime | Single language for CLI + viewer + MCP. Compiles to native binary. Founder knows some JS, and end users never write code (NL + YAML + SQL). |
| A2 | Embed viewer assets in binary | Single-file distribution (~8-15MB with Bun runtime + ECharts). `curl\|sh` installs just work. Zero runtime deps. Fallback: if Bun compile cannot embed static assets, use thin wrapper that unpacks assets to temp dir on first run. Validate during Phase 0 spike. |
| A3 | Single process, dual protocol | Bun binary serves HTTP (web viewer) and stdio (MCP) simultaneously. Shared core library. |
| A4 | Read-only DB connections | Prevents write operations via LLM-generated queries. Setup guide instructs users to create a read-only DB user/role. Defense-in-depth: also validate generated SQL is SELECT-only via lightweight parser. |
| A5 | Env var references for secrets | Specs reference `$DATABASE_URL`, not raw connection strings. Passwords never committed to git. |
| A6 | LLM structured output (tool_use) | LLM returns a JSON object via tool_use; CLI serializes it to YAML for human editability. Eliminates YAML syntax errors. |
| A7 | Server-side aggregation | Bar/line/pie/kpi charts aggregate on DB side (GROUP BY). Only send summarized data to browser. Tables paginated at 50 rows. |
| A8 | Connection pool limit: 4 concurrent | Prevents pool exhaustion when dashboards have many charts. Queue remainder with per-chart spinners. |
| A9 | 3-tier test strategy | Tier 1: ~50 unit tests (deterministic). Tier 2: ~20 integration tests (Playwright). Tier 3: ~10 LLM eval golden specs (~$0.50/run). |
| A10 | Query profiling mode | `dashcli serve --profile` shows execution time badge + SQL tooltip per chart. |
| A11 | Prompt injection mitigation | Defense-in-depth: (1) regex deny-list strips patterns like "ignore previous instructions", "system:", etc. (2) System prompt boundary markers. (3) Output validation: verify generated SQL is SELECT-only via lightweight parser (second layer beyond A4). |
| A12 | In-flight query cancellation | Filter changes cancel running queries before issuing new ones. Prevents stale data races. |

## Error & Rescue Summary

13 error paths mapped across 5 subsystems. 4 critical gaps found and fixed during review:
- LLMParseError: log raw response, retry once, save partial spec
- LLMRefusalError: prompt user to rephrase
- LLMSchemaError: strip unknown keys, warn
- PoolExhaustedError: queue with max 4 concurrent queries

All 13 failure modes now have: rescue action, user-facing message, logging, and planned test coverage. Full error registry in the design doc at `~/.gstack/projects/claude-tmp/qikeli-master-design-20260322-114848.md` and in the CEO review session (Section 2).

## Security Mitigations

| Threat | Mitigation |
|--------|-----------|
| SQL injection (LLM queries) | Read-only DB connections (A4) |
| SQL injection (filter values) | Parameterized queries (from design doc) |
| Connection string secrets in git | Env var references (A5) |
| LLM prompt injection | Defense-in-depth: input sanitization + prompt hardening + SELECT-only SQL validation (A11) |
| Export path traversal | Input validation on `--output` flag |

## Interaction Edge Cases (from Section 4)

5 viewer edge cases resolved:
1. Zero charts → empty state message
2. All queries fail → per-chart error (not full-page crash)
3. Filter change during in-flight queries → cancel + re-query (A12)
4. Browser resize → wire ECharts resize handlers
5. Special chars in export filename → sanitize

## Phase Structure

- **Phase 0** — Validation prototype (CSV + 3 charts). Show Nina (data scientist on the team, primary target user). Success = she asks to use it on real data.
- **Phase 1 (MVP)** — Full CLI (create, edit, serve, export, watch) + web viewer (6 charts, themes, profiling) + MCP server + Postgres/CSV connectors + Smart Schema Summary. Effort: human ~6 weeks / CC (Claude Code agent hours) ~10-18 hours (base ~8-16h + ~2-4h for accepted scope additions: schema summary, theme, watch). Note: 6 weeks assumes heavy CC delegation; break into sub-milestones during /plan-eng-review.
- **Phase 2** — `dashcli suggest`, `dashcli init`, more data sources, more chart types, `dashcli publish`, query caching.
- **Phase 3** (contingent on Phase 1-2 traction) — Auth, sharing, collaboration, scheduled refresh. Hosted SaaS is exploratory and implies a different product/infrastructure — evaluate after Phase 2 metrics.

**Note:** The 10x vision (autonomous data analyst, proactive discovery, dashboard retirement) is a North Star for Phase 4+. Phases 1-3 intentionally do not pursue autonomy or proactive behavior.
